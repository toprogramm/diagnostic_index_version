# INDEX/VERSION MONITOR

Простой и эффективный инструмент для мониторинга состояния сервисов и контроля версий на нескольких серверах через единый веб-интерфейс.

## Основные возможности

- **Мониторинг статуса (Index)**: Отслеживание состояния ключевых компонентов сервера.
- **Контроль версий**: Сравнение версий сервисов с целевой версией (указанной в футере страницы).
- **Визуальная индикация**:
  - <span style="color: green">↑ Зеленая стрелка</span>: версия выше целевой.
  - <span style="color: red">↓ Красная стрелка</span>: версия ниже целевой.
- **Валидация конфигурации**: Автоматическая проверка `config.json` на дубликаты ID, имен и URL при запуске.
- **Поддержка Production**: Готовность к развертыванию через промышленный сервер `waitress`.

## Настройка

### 1. Переменные окружения (`.env`)
Создайте файл `.env` (используя `.env.example` как образец) и настройте следующие параметры:

| Переменная | Описание | Значение по умолчанию |
| :--- | :--- | :--- |
| `LOGIN` / `PASSWORD` | Учетные данные для доступа к серверам | - |
| `PROD` | Режим работы (`true` — промышленный, `false` — отладка) | `false` |
| `HOST` | IP-адрес для прослушивания сервером | `0.0.0.0` |
| `PORT` | Порт приложения | `5000` |
| `SESSION_TIMEOUT` | Таймаут авторизации (в секундах) | `10` |
| `REQUEST_TIMEOUT` | Таймаут основных запросов (в секундах) | `300` |
| `SIGNIN_TAIL` | Путь к странице авторизации | - |
| `VERSION_TAIL` | Путь к странице версий | - |
| `INDEX_TAIL` | Путь к главной странице сервера | - |
| `PAGE_PARSER_BODY` | ID контейнера для парсинга данных | - |

### 2. Конфигурация серверов (`config.json`)
Опишите список серверов в файле `config.json`:

```json
{
  "servers": [
    {
      "id": "srv-01",
      "name": "Основной сервер",
      "url": "https://server1.example.com"
    }
  ]
}
```

## Установка и запуск

### Стандартный запуск (с доступом к сети)

1. **Подготовка окружения:**
   ```bash
   python -m venv venv
   # Активация:
   # Windows: venv\Scripts\activate
   # Linux: source venv/bin/activate
   ```

2. **Установка зависимостей:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Запуск:**
   ```bash
   python app.py
   ```
   *Приложение будет доступно по адресу, указанному в `HOST` и `PORT`.*

### Offline-установка (без доступа к сети)

1. **На машине с интернетом:**
   ```bash
   pip download -d packages -r requirements.txt
   ```
2. **Перенесите проект и папку `packages` на целевую машину.**
3. **На целевой машине:**
   ```bash
   python -m venv venv
   # Активируйте venv
   pip install --no-index --find-links=packages -r requirements.txt
   python app.py
   ```

## Требования
- Python 3.8+
- Библиотеки: Flask, requests, beautifulsoup4, waitress, python-dotenv.
